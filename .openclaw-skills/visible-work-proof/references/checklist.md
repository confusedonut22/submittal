# Visible Work Proof Checklist

## Before claiming visible work
- confirm the tool/harness actually started
- if it failed, report the failure immediately
- switch to a direct fallback path quickly

## Progress update format
- current file being changed
- last command run
- last commit / test result

## Avoid
- vague reassurance without evidence
- implying "running" when the command failed
- long silent gaps when the user explicitly asked for visibility
